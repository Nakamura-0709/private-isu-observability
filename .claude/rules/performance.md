# Performance Tuning Rules

## 計測データなしの改善提案禁止

改善提案を行う前に、必ず以下のいずれかの計測データを確認すること。

- `make bench` のスコアと変化率
- `alp` によるエンドポイント別レスポンスタイム（Sum, Avg, Max）
- `pt-query-digest` によるスロークエリ（Query_time, Rows_examined）
- Grafanaダッシュボードのメトリクス
- Pyroscopeのフレームグラフ
- OTelのトレース（スパン単位の遅延）

計測データが手元にない場合は、`make bench && make analyze` の実行を先に促すこと。

## 変更の単位

- 1つの変更ごとに必ず `make bench` を実行し、スコアとメトリクスの変化を記録すること。
- 複数の変更を同時に行わない。効果の切り分けができなくなる。
- 変更後にスコアが下がった場合はすぐにリバートし、原因を特定してから再試行すること。

## 改善ログの記録

効果が確認できた変更は `/record-improvement` で `.claude/history.md` に記録すること。
過去に試して失敗した施策を繰り返さないための記憶として活用する。
