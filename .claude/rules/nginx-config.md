# Nginx Config Rules

## 変更前の確認

- `alp` でNginxのアクセスログを分析し、改善対象のエンドポイントを特定してから設定を変更すること。
- 変更前のベンチマークスコアを記録しておくこと。

## 構文チェックの必須化

設定ファイルを変更したら、デプロイ前に必ず構文チェックを実施すること。

```bash
nginx -t
```

エラーがある場合はデプロイしないこと。

## 変更後の検証

- `make deploy` 後に `make bench` を実行し、変更前後のスコアを比較すること。
- `alp` で再度ログを分析し、レスポンスタイムの改善を定量的に確認すること。

## よく使う最適化の注意点

- `worker_processes`: CPU コア数に合わせて設定する（`auto` 推奨）。
- `keepalive_timeout`: 長くしすぎるとコネクション枯渇につながる。計測データで確認すること。
- 静的ファイルの `expires` 設定はベンチマーカーのキャッシュ挙動を確認してから行うこと。
